# MoodTune AI — 情绪音乐推荐工具

> 输入心情 → AI 推荐 3 首专属歌曲 → 一键跳转网易云/QQ 音乐收听。

> 截图占位：`screenshots/app.png` — 完整推荐页面（Hero + 3 首歌曲卡片 + 反馈区）【需手动截图放置】

---

## 解决的真正问题

想听歌但不知道听什么、网易云日推不准、心情不好时 AI 歌单往往比人工推荐更懂你。MoodTune 不依赖播放历史，只根据你此刻的情绪状态生成专属推荐。

---

## 核心设计决策

### 决策 1：快速模式 vs 标准模式的双模设计

| | 快速模式 | 标准模式 |
|---|---|---|
| System Prompt 长度 | 60 字 | 120 字 |
| max_tokens | 320 | 500 |
| temperature | 0.3 | 0.5 |
| 生成速度 | 3-8 秒 | 8-20 秒 |
| 推荐质量 | 简洁精准 | 更丰富 |

**设计思路**：默认开启快速模式（短 Prompt + 低 token）。用户的第一需求是"快来一首对的歌"，而不是"给一篇乐评"。快速模式通过限制输出长度和降低随机性来保证速度和稳定性。

### 决策 2：反馈闭环 —— 点踩直接改变下次推荐

不同于"收集反馈,以后优化"的常规做法，MoodTune 的点踩**即时生效**：

```
用户点踩《XX》→ 下一轮 Prompt 追加「请避免推荐《XX》或风格过近的歌」
→ 再点"更激昂"微调 → Prompt 再追加「推荐更激昂、更有力量感的歌曲」
```

每次交互都在**累积上下文**（不喜欢列表 + 微调偏好），prompt 越来越精准。这个设计把一个"用完即走"的推荐工具变成了"越用越懂你"的产品。

### 决策 3：Structured Input — 不止是文本框

不同于 ChatGPT 的自由文本输入，MoodTune 提供了结构化情绪输入：

```
情绪标签（多选）+ 场景（通勤/运动/睡前/学习/工作）+ 能量值（1-10）+ 用户原话（可选）
```

**原因**：纯文本输入对用户要求高（"我不知道怎么描述我的心情"），结构化选项降低输入门槛的同时，为 LLM 提供了可解析的情绪上下文。这几项拼成一句话传入 prompt，比纯文本更可控。

### 决策 4：为什么不展示原始 JSON？

MVP 初期用 `st.json()` 直接展示模型输出，方便调试但体验差。改成 Spotify 风格深色卡片 UI：

- 渐变 Hero 顶栏 + 品牌徽章
- 歌曲卡片（封面色块/专辑封面 + 歌名/歌手 + AI 推荐理由 blockquote + 悬停动效）
- 网易云/QQ 音乐一键跳转按钮
- 分享卡片生成（PIL 渲染 PNG）

**产品直觉**：AI 产品如果输出看起来像"开发中"的 JSON，用户不会信任推荐质量。好的 UI 包装能显著提升产品的感知质量和可信度。

## 反馈闭环设计

```
心情输入 → AI 推荐 3 首 → 歌曲卡片展示
    │                        │
    │                        ├── 单曲点赞 👍 → 记录偏好
    │                        ├── 单曲点踩 👎 → 作为负面示例注入下轮 prompt
    │                        └── 全局评价 → 记录满意度
    │
    ├── 🔥 更激昂 → prompt 追加激昂偏好 → 二次推荐
    ├── 🌙 更安静 → prompt 追加安静偏好 → 二次推荐
    ├── 🔄 换一批 → 排除已推荐 + 点踩歌曲,重新生成
    └── ➕ 再推几首 → 追加推荐（不替换,去重）
    │
    ▼
心情日记 → SQLite 持久化 → 情绪趋势看板 + AI 心情洞察
```

---

## 情绪趋势

MoodTune 不只是单次推荐工具。每次生成歌单后自动记录到 SQLite，形成长期情绪数据：

- 能量值折线图（按日/周/月）
- 情绪标签分布柱状图
- AI 心情洞察（基于近期标签频次 + 日记摘要生成温柔的心理陪伴式解读）

## 本地运行

```bash
pip install -r requirements.txt
streamlit run app.py
```

Windows 也可双击 `start.bat`。

**DeepSeek 配置**（侧边栏填写）：
- Base URL: `https://api.deepseek.com`
- 模型: `deepseek-chat`
- 快速模式: 开启
- 超时: 90-180 秒

---

## 技术架构

```
Streamlit 前端
    │
    ├── 侧边栏 API 配置（Key/Base URL/模型/超时/快速模式）
    ├── 心情输入区（标签/场景/能量/原话）
    ├── LLM 层（DeepSeek API · OpenAI 兼容 · 流式输出）
    ├── 结果展示（CSS 深色主题 · 歌曲卡片 · 专辑封面）
    ├── 反馈层（单曲点赞踩 · 全局评价 · 一键微调）
    └── 持久化层（SQLite · 推荐历史 · 心情日记 · 趋势看板）
```

- **前端**：Streamlit + 自定义 CSS（深色 Spotify 风格）
- **AI**：DeepSeek chat（OpenAI SDK 兼容）
- **数据**：SQLite（设备 ID 隔离用户）
- **专辑封面**：Deezer API
