# MoodTune AI — Prompt 实验记录

> 日期：2026-06-16 | 核心问题：Fast 模式是否牺牲了推荐质量？

---

## 一、实验背景

MoodTune 有两种生成模式：

| 模式 | System Prompt | max_tokens | temperature | 生成速度 |
|------|--------------|------------|-------------|----------|
| Fast | 60 字 | 320 | 0.3 | 3-8 秒 |
| Standard | 120 字 | 500 | 0.5 | 8-20 秒 |

**默认选择 Fast 模式的原因**：用户等 8 秒以上会焦虑，而音乐推荐不需要"深度推理"，需要的是快速 + 精准。

**需要验证的问题**：Fast 模式在缩减 Prompt 和 token 的前提下，推荐质量是否明显下降？

## 四、初步观察（定性）

**Fast 模式的优势**：
- 因为 temperature 更低（0.3 vs 0.5），输出更稳定可控
- 短 Prompt 减少了 LLM "想太多"导致推荐偏离用户心情的概率
- 速度优势明显（3-8s vs 8-20s），对移动场景更友好

**Standard 模式的优势**：
- 推荐理由更丰富、更个性化（像真人在推荐，而非模板）
- 歌曲多样性略高（token 预算允许更详细的推荐说明）

**初步结论**：Fast 模式在多数日常场景下不输 Standard。Standard 更适合情绪复杂或用户想"深度体验"的场景。

---

## 五、设计启示

1. **对 AI 产品 Prompt 设计的启发**：不是越长越好。System Prompt 的核心信息密度比字数更重要。Fast Prompt 在 60 字内完成了"角色 + 输出格式 + 示例"三个关键要素。
2. **Temperature 的选择**：音乐推荐这种主观任务，temperature 0.3 已经足够。0.5 虽然多样性高，但也增加了"出戏"的风险（推荐了完全不搭的歌）。
3. **用户等待与质量感知的权衡**：在推荐领域，3 秒出结果给用户的体验感知，比多等 10 秒获得"稍微好一点"的推荐更好。

---

## 六、如何跑评测

在 MoodTune 侧边栏：
1. 关闭快速模式 → Standard 模式跑一次
2. 开启快速模式 → Fast 模式跑一次
3. 对比两次结果，人工打分

每次评测约 2 分钟。10 个场景约需 20 分钟完成全部对比。
